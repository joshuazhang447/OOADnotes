Week 4: Software Development Fundamentals

The Nature of Software Development

A core principle in software development is that software is never static; it always evolves and changes. This constant evolution is a driving force behind the methodologies used to create and maintain software.

Key Development Stages

  • Business Modeling (Domain Model): This initial phase focuses on understanding the business model and the problem domain.
  • Requirements: This stage involves defining what types of functional requirements the system must fulfill.
  • Design: The focus then shifts to designing the system's architecture and components.

System Boundaries

Understanding the scope of a system is crucial. System boundaries define what is considered part of the system ("In") and what is external to it ("Out"). This helps in clarifying the interactions between different components. A system boundary is often visualized as a rectangle in use-case diagrams that separates internal use cases from external actors.

  • Actor: An entity that interacts with the system from the outside.
  • System: The components and processes that are internal to what is being built.
  • Goal: The objective that the actor aims to achieve by using the system.

Evolving Requirements: A Dog Door Example

To illustrate how software requirements evolve, consider the example of an automated dog door.

Actors

  1. Todd / Gina (the owners)
  2. Dog (Fido)

Old Requirements (Manual Process)

  1. Fido Barks.
  2. Todd/Gina hears the barks.
  3. The dog door opens (manually).
  4. Fido is out doing his business.
  5. The door shuts automatically.
  6. Fido barks to be let back in.
  7. Todd/Gina hears Fido barking.
  8. The door is opened.
  9. Fido goes inside.

New Requirements (Automated System)

The new requirements aim to automate the process, reducing the need for human intervention.

  1. The door automatically opens when it hears a bark (utilizing a bark recognition sensor).
  2. A button can be pressed on a remote to open the door.
  3. The system uses face recognition to identify the dog.
  4. An RFID tag on the dog's collar can trigger the door to open.
  5. The door shuts automatically.
  6. To get back in, the owner can press a button.
