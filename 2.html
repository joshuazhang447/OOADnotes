Week 2 Notes: UML & Class Design

Introduction to UML

UML ⇒ Universal Modelling Language.

UML is a standardized modeling language used to visualize, specify, construct, and document the artifacts of a software system. Key diagram types include:

  • Class Diagram
  • Sequence Diagram
  • Use Case Diagram

Class Design Review Questions

Questions to consider for improving the Guitar class design:

  1. Do you think you have all the properties needed for your Guitar Class?
  2. Do you have all your methods?
  3. What would you change/add to it to make it better?
  4. What is the first thing you would change?
  5. Did you think about using encapsulation in your code? If yes, why? Explain. If no, why? Explain.

3 Steps to a Great Software

Key principles to keep in mind when developing software:

  • Customer needs / satisfaction: The software must solve the customer's problem effectively.
  • Apply SOLID Object-Oriented principles: Following established design principles leads to better software.
  • Maintainable and reusable design: The software should be easy to update, fix, and extend over time.
